# wsgi应用(解释器)
这个层次是逻辑业务层，flask，fastapi，django，他们的最终都是产生一个符合wsgi规范的web应用，可以处理请求，业务逻辑。
flask的create_app()应用工厂返回的就是一个wsgi应用。

# WSGI服务器(进程管理器)
gunicorn这种WSGI服务器主要作用就是为上面的应用提供性能和稳定。实现了业务逻辑和服务器进程管理的分离。
## http请求转换
把从nginx那边来的过滤过一波的http请求，转换成符合wsgi协议的environ字典，让flask能处理，他也能接收flask的wsgi协议的environ字典转换成http请求发送回去。
## 多进程并行
如下命令，创建了四个进程(worker)，每个进程都是一个flask app。从而避免python的GIL(全局解释锁)。
```bash
 uv run gunicorn -w 4 -b 127.0.0.1:5000 "app:create_app()"
 ```
 ## 提高稳健性
 如果是上面的启动参数gunicorn的主进程会监控这4个worker，一但worker挂了，他会主动杀掉他重新创建一个新的worker，提高服务器的稳健性

 # Nginx
 单纯一个gunicorn其实已经可以用了，但是我们可以让权责分离的更细致，让python的web服务器只处理逻辑业务，让nginx先过滤一遍请求，需要业务逻辑处理的再交给gunicorn
 ## SSL
 nginx处理HTTPS加密优化很好，这个部分让gunicorn做效率太低。
 ## 缓冲
 如果gunicorn处理请求的速度慢了，可以先缓存到nginx等他慢慢处理。所以它可以抗大量请求的并发
 ## 静态资源
 nginx响应静态资源非常快，但是用python的http.server性能就很差，我们尽量只让python来处理复杂的逻辑业务。
## 负载均衡

这个多端结构的优势能让各个层次各司其职，利用各个的优势，提高系统的性能。让业务和运维分离。
